Overview

A lightweight, client-side notes sharing platform. Anyone can upload notes (Google Drive link or name), browse subject-wise, search, paginate through the vault, and file complaints for incorrect or inappropriate uploads. Admins review complaints and manage notes from a separate admin panel.

  • Upload notes with Sub Code, Sub Name, Topic/Module, and Drive link/name.
  • Browse subject-wise and search across sub code, sub name, topic, and notes name.
  • Pagination to navigate large lists of notes.
  • Complaints allow users to request deletion; admins review and act.
  • Storage uses Local Storage (client-side demo).
Open Notes Vault

1. Upload

Open the vault and click Upload Note. Fill Sub Code, Sub Name, Topic/Module, and Notes Drive Link or Name. Uploads are anonymous and stored locally.

2. Browse & Search

Use the search box and subject filter to find notes quickly. Results are paginated for easy navigation.

3. Complaints & Admin

If a note is wrong or inappropriate, file a complaint. Admins (separate admin page) review complaints and can delete notes. Admin credentials (demo): Admin / Admin1010.

Key features

Subject-wise organization

Notes are tagged with Sub Code and Sub Name so you can quickly filter by subject.

Simple uploads

No account required to upload. Fill the popup form and submit — the note appears in the vault immediately.

Complaint workflow

Users can flag notes for admin review. Complaints are stored in Local Storage for admin action.

Client-side demo

All data is stored in the browser's Local Storage. For production, migrate to a backend for secure auth and persistent storage.
